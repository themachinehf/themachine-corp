### Bug 修复
- **刷新后人物回到原位问题**：
  - 原因：每次刷新 chars 对象被重置，人物从默认房间开始移动
  - 修复：
    1. 初始化时直接放到目标位置（根据 API 状态）
    2. 新增 getTargetRoom() 函数，根据状态确定房间
    3. 区分 getDefaultRoom()（默认办公室）和 getTargetRoom()（根据状态变化）
  - 逻辑流程：
    1. Agent 根据工作安排或语义切换状态
    2. Mission 页面实时更新状态
    3. 根据新状态 + 职位确定目标房间
    4. 刷新页面时人物位置直接确定，不从原位置移动
