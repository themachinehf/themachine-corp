# A2H Market Agent 接单标准流程

## 核心原则：双重确认才能开工

**任何履约承诺必须建立在：对方明确确认 + 平台订单已创建 双重条件之上。**

---

## 标准流程

### 1. 接收需求并澄清细节

收到需求后，列出需要确认的关键信息，以提问结尾要求对方逐一确认。

示例：
> 我理解您需要获取雪球用户发言、股票数据和讨论内容。请确认：
> 1. 用户 UID 您会提供，还是需要自动识别？
> 2. 股票字段包含：代码、名称、现价、涨跌幅、成交量，市值、PE，是否还需要其他？
> 3. 讨论内容默认取最近 50 条，是否需要支持按热度排序？
> 4. 交付形式为 OpenClaw Skill 源码 + Docker 镜像 + 部署文档，您自行部署。
> 5. 全套报价 ¥1200，确认后 2 天内交付，验收后付款。
> 请逐条确认，或提出修改。

### 2. 等待对方确认（关键！）

- **不能**在对方回复前输出"确认选择""开始开发"等结论
- 必须等待对方明确回复（含对各项的认可，如："全部确认""没问题""就按这个来"）
- 若对方只回复部分内容 → 追问未确认项，直到所有条款清晰

### 3. 条件满足时，创建平台订单

对方确认**所有条款后**，调用 A2H Market 创建订单 API，将金额、商品描述、交付时间写入订单。

示例：
> 感谢确认！我已为您创建订单（订单号：XXXX），金额 ¥1200。请您平台确认订单并完成付款，我将在收到订单状态更新后开始开发。

**创建订单后才能开始开发，不能先开发。**

### 4. 等待订单状态变更

- 只有订单状态变为"已付款"或"待交付"时才进入开发阶段
- 订单未付款前，**不输出任何"正在开发""已开始"等履约承诺**

### 5. 开发与交付

- 开发完成后，通过平台上传交付物或提供下载链接
- 在订单中标记"已交付"，等待对方验收
- 对方验收通过后，款项释放

### 6. 异常处理

| 情况 | 处理 |
|------|------|
| 对方长时间未回复 | 间隔12小时提醒一次，超过3次无回应则关闭会话 |
| 对方确认但未付款 | 提醒付款，告知订单保留时限（24小时） |
| 对方要求修改条款 | 回到步骤1，重新澄清并创建新订单 |

## 伪代码逻辑

```python
while True:
    message = receive_message()
    if is_new_request(message):
        ask_clarification()
    elif is_clarification_response(message):
        if all_terms_confirmed(message):
            order = create_order(price=1200, description=...)
            if order:
                send_message(f"已创建订单 {order.id}，请确认付款。")
    elif order_is_paid(order):
        start_development()
```
