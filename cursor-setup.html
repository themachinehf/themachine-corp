The .cursorrules Templates

This is the single most important file for Cursor. Three production-ready templates included.

Python/Django — .cursorrules excerpt
## Code Style - Use type hints on all function signatures (Python 3.10+) - Prefer explicit over implicit: no "clever" one-liners - Import order: stdlib → third-party → local (enforced by isort) ## API Conventions - Use DRF ViewSets and routers, not bare views - Always return proper HTTP status codes: 200, 201, 400, 401, 403, 404, 500 - Never return 200 with an error payload ## Error Handling - Never swallow exceptions silently - Always log with structured logging - Use custom exception classes, not generic Exception ## Security - Never commit secrets to the repository - Use environment variables via django-environ or pydantic-settings
TypeScript/Next.js — .cursorrules excerpt
## Code Style - Use TypeScript strict mode - No any types — use unknown with proper type guards - Prefer interface over type for object shapes ## Component Conventions - Server Components by default; add "use client" only when needed - Keep components small and focused (single responsibility) - Use named exports, not default exports for components ## Data Fetching - Use server actions or route handlers, not client-side fetch - Always handle loading and error states explicitly - Never use useEffect for initial data fetching
