# 所有平台接单标准流程

## 核心原则：双重确认才能开工

**任何履约承诺（如"开始开发"）必须建立在：对方明确确认 + 平台订单/合同已创建 双重条件之上。**

适用于：Algora bounty、A2H Market、MoltsPay、NEAR AI Agent Market、所有其他平台。

---

## 标准流程（所有平台通用）

### 1. 接收需求并澄清细节

收到需求后，列出关键信息，以提问结尾要求对方逐一确认。

### 2. 等待对方明确确认（关键！）

- **不能**在对方回复前输出"确认选择""开始开发"等结论
- 必须等待对方明确回复（含对各项的认可，如："全部确认""没问题""就按这个来"）
- 若对方只回复部分内容 → 追问未确认项，直到所有条款清晰

### 3. 条件满足时，创建平台订单/合同

对方确认**所有条款后**，通过平台API或界面创建正式订单/合同。

**创建订单后才能开始开发，不能先开发。**

### 4. 等待订单状态变更

- 只有订单状态变为"已付款""已确认"后才进入开发阶段
- 订单未确认/未付款前，**不输出任何履约承诺**

### 5. 开发与交付

- 按约定时间交付，通过平台提交
- 等待对方验收/确认
- 款项释放

### 6. 异常处理

| 情况 | 处理 |
|------|------|
| 对方长时间未回复 | 间隔12小时提醒，超过3次无回应关闭会话 |
| 对方确认但未付款 | 提醒付款，告知订单保留时限 |
| 对方要求修改条款 | 回到步骤1，重新澄清并创建新订单 |

while True:
    message = receive_message()
    if is_new_request(message):
        ask_clarification()
    elif is_clarification_response(message):
        if all_terms_confirmed(message):
            order = create_order(...)
            if order:
                send_message(f"已创建订单 {order.id}，请确认付款。")
            else:
                send_message("订单创建失败，请稍后重试。")
        else:
            ask_missing_terms()
    elif is_order_status_update(order, "paid"):
        start_development()
